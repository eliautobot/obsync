# Obsync v0.12.0

Obsync v0.12.0 turns Local AI into a configurable Obsidian record keeper. Complete document transfer is now the default, every model-facing setting is inspectable, and reusable AI profiles control both inference and the resulting vault structure.

## Highlights

- **Correct live version:** The sidebar reads the running server version from `/api/v1/meta`; it is no longer a hardcoded stale label.
- **Full document transfer by default:** The complete extracted source text is written directly into the generated note. Model summaries add search context but cannot replace or abbreviate the document body.
- **Built-in profiles:** Immutable **Full document transfer** and **Brief summary** profiles provide dependable fallbacks.
- **Custom profiles:** Copy either built-in, then rename, edit, save, activate, or delete the custom profile. Built-ins cannot be edited or deleted.
- **Complete inference controls:** The Local AI page exposes the role prompt, user-prompt template, content behavior, temperature, Top P, maximum output tokens, input character limit, vault candidate limit, tag limit, and related-note limit.
- **Visible protected boundary:** The safety and structured-output prompt is inspectable but read-only, so the model contract is transparent without allowing a custom profile to remove the prompt-injection guard.
- **Obsidian behavior controls:** Each custom profile can enable or disable vault context, `[[wikilinks]]`, tags, YAML properties, category folders, and source details.
- **Real vault context:** Server-mounted and Desktop-hosted vaults are cataloged for Markdown title, relative path, and frontmatter tags. Relevant bounded candidates are supplied to the model, and returned links are accepted only when they exactly match the catalog.
- **Provider parameters honored:** Ollama and OpenAI-compatible requests now receive the active profile's sampling and output-token parameters.

## Obsidian integration model

Obsidian does not expose a remote core API for this workflow. Obsync integrates with the vault's native contract instead: Markdown files, YAML properties, tags, folders, and `[[wikilinks]]`. The model never receives arbitrary vault or filesystem access. Obsync chooses bounded context, validates the model's structured response, and applies the enabled features itself.

## Upgrade behavior

1. Back up the Obsync data volume and Obsidian vault.
2. Update the server to v0.12.0 and confirm `/api/v1/health` and the sidebar both report `0.12.0`.
3. Update Obsync Desktop to v0.12.0.
4. Open **Local AI**, confirm the desired active profile, and run a representative document through Review before enabling a large batch.

Database schema version 7 adds custom AI profile storage and a bounded vault-note catalog. Existing accounts, pairings, watched folders, documents, reviews, model connection settings, and vault selections are preserved. Legacy custom AI instructions are retained as an inactive **Imported AI settings** custom profile, while **Full document transfer** becomes active so upgraded installations stop producing summary-only notes by default.

## Validation

- Automated unit and integration tests cover profile CRUD, immutable built-ins, activation fallback, legacy migration, prompt validation, provider parameters, full/summary Markdown behavior, vault tag parsing, and Desktop vault catalog updates.
- Browser testing covers profile copy/edit/save/activate, read-only protected prompts, dynamic version display, desktop layout, a 390-pixel mobile viewport, horizontal overflow, and console errors.
- Release gates include linting, formatting, branch coverage, JavaScript syntax, package builds, Docker health/auth/profile checks, Windows and Linux CI, release artifact hashes, and exact-image verification.

## Safety

This release does not change the source-file boundary: Obsync never edits, moves, or deletes source documents. Non-Obsync destination collisions still stop processing, managed-note writes remain atomic, and content below **My notes** remains preserved.
