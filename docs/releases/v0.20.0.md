# Obsync v0.20.0

Obsync v0.20.0 makes Maintenance graph-first in practice. The model no longer receives a candidate
list and invents a connection in one step. It must first extract grounded factual claims, then final
link selection can only copy a supported edge already stored for that source note.

## Persistent factual graph

The private vault graph now persists:

- Heading-aware Markdown chunks
- Canonical documents, durable entities, aliases, and exact mentions
- Directional typed factual edges
- Exact source quotes, character offsets, chunk IDs, and content hashes
- Confidence and explicit temporal state
- Folder hierarchy, entity document frequency, and learned Review weights

Index Sweep remains completely read-only. It builds the deterministic document, structure, chunk,
and exact-reference layers without calling Local AI. Maintenance performs the factual extraction
pass, reuses still-valid claims for unchanged note hashes, recalculates entity rarity, and then runs
relationship selection.

## Supported edges only

Each inline link must copy an exact precomputed edge ID with its canonical endpoints and predicate.
Similarity and shared metadata can retrieve a candidate, but neither can create a link. Generic or
invented predicates, missing provenance, ambiguous target documents, and common-entity fan-out are
discarded.

The anchor must be a complete natural entity or document phrase already present in the source.
Fragments such as `who want a more`, `define exact Free vs Pro`, and `around Virtual Office` are
rejected. The default safety ceiling is now three inline links per note.

Ordinary reverse links are suppressed because Obsidian already exposes them as backlinks. Exact
category hubs remain eligible when the source names the hub and the hub directly catalogs the
source; that relationship is stored directionally as `cataloged_by`.

## Safe legacy cleanup and adaptive reviews

Old marked maintenance blocks remain deterministic cleanup, not an AI judgment. If a legacy block
contains tags that exist nowhere else in the note, Obsync first proposes native YAML additions.
The cleanup operation cannot be applied unless those tags already exist or their migrations are
selected in the same review. Existing human YAML tags are never removed by this migration.

Approvals and rejections update bounded weights for predicates, anchor shapes, tag values, and
operation kinds. Feedback influences later confidence only after repeated reviews; it cannot bypass
the evidence and graph gates.

Maintenance progress now updates recommendation and changed-note totals while the sweep is still
running, and the final values match the Review queue.

## Upgrade behavior

Database schema 14 adds the persistent graph, hierarchy, extraction-state, and feedback tables. It
supersedes pending recommendations made by the previous engine, invalidates the cached learned
model, and reduces installations using the former eight-link default to three. Existing Markdown
is unchanged during migration.

After updating, run a full Index Sweep, then run Maintenance in Review mode. Do not enable
Automatic mode until the recommendations in your own vault are consistently correct.

Back up both the Obsync `/data` volume and the Obsidian vault before upgrading. To roll back,
restore the pre-upgrade data backup together with the v0.19.0 application or container; do not run
v0.19.0 against a database already migrated to schema 14.

## Verification

The release passed 264 automated tests with 83 percent branch coverage. A disposable 511-note
Index → Maintenance end-to-end benchmark processed every note with zero decision failures, zero
unsafe anchors, both expected exact links, preserved legacy tags, reciprocal suppression, and no
Review-mode Markdown writes. SQLite integrity and edge provenance checks passed, and selective
apply followed by Undo restored the complete vault byte for byte.

The GitHub release includes Linux and Windows command-line agents, Obsync Desktop for Windows, and
`SHA256SUMS.txt` for download verification. The matching server image is published as
`ghcr.io/eliautobot/obsync:0.20.0`.
