# Obsync v0.13.1

Obsync v0.13.1 is a focused reliability update for whole-vault sweeps introduced in v0.13.0.

## Fixed

- YAML dates and timestamps are converted to stable ISO 8601 strings before a Desktop vault batch is sent to the server.
- Nested YAML maps and lists, date-valued keys, non-finite numbers, binary values, and other uncommon YAML-native objects are normalized into bounded JSON-safe values.
- Index and Maintenance Sweeps no longer stop when an otherwise valid Obsidian note contains an unquoted YAML date.
- A failed sweep now leaves a visible error beside its controls and shows the complete error in sweep history instead of only removing the active progress bar.
- The Vault page now distinguishes the current cached-note count from the last successful Index Sweep.

## Safety

This patch does not change source files, linking rules, review behavior, sweep schedules, or vault write permissions. It only normalizes parsed frontmatter metadata at the Desktop serialization boundary. Complete note content remains unchanged in the vault index.

## Validation

Validation completed before release:

- 141 automated tests passed with 80% branch coverage.
- Regression coverage exercises the real paired Desktop upload path with YAML dates and timestamps.
- Strict JSON coverage includes nested maps and lists, date-valued keys, sets, binary values, and non-finite numbers.
- A container smoke test completed a full Index Sweep against a dated Obsidian note.
- Browser verification confirmed the persistent Index and Maintenance failure alerts, clarified cache wording, complete history errors, and a clean console.
