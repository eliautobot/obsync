# Obsync v0.14.0

Obsync v0.14.0 replaces broad deterministic relationship linking with an adaptive, evidence-gated Local AI engine learned separately for each Obsidian vault.

## Why this release exists

The v0.13 maintenance matcher could treat shared vocabulary as a relationship. Repeated records such as invoices often use the same template, folder, tags, and terms, so the matcher could connect an invoice to many unrelated invoices. Those similarities are useful for candidate retrieval but are not proof of a meaningful knowledge-graph relationship.

v0.14 separates those responsibilities. Retrieval may shortlist a similar note, but only Local AI can propose a link, and Obsync accepts it only after strict validation.

## Adaptive per-vault organization

- Obsync learns a vault-specific organization model from indexed note content, folders, headings, aliases, properties, tags, and structure.
- The model learns the vault's own note roles, organization principles, relationship guidance, negative guidance, and folder conventions.
- No business names, document categories, relationship types, or folder layouts are compiled into the relationship engine.
- Applied and rejected Review decisions become feedback for the next model refresh.
- The model automatically refreshes when vault knowledge, the active AI profile, Local AI provider/model, or review feedback changes.

## Evidence-gated relationships

Every accepted relationship must have:

1. An exact path-qualified target from the current vault candidate allowlist.
2. A specific factual description of how the two records relate.
3. A `SOURCE:` fact grounded in the source note.
4. A `TARGET:` fact grounded in the target note.
5. Confidence at or above the configured threshold.

A shared word, folder, tag, template, category, or document type is never sufficient. Empty relationship decisions are valid and preferred when the evidence is weak.

## Maintenance repair and migration

- Generated maintenance blocks are excluded from all future indexing and inference evidence, preventing self-reinforcing link growth.
- A new Maintenance Sweep recalculates generated relationship blocks with the adaptive engine.
- If the AI finds no supported relationship, Review can remove the old generated block while preserving every byte outside Obsync's marked block.
- If Local AI fails, the note is skipped and its current block is preserved.
- Pending v0.13 deterministic recommendations are marked `superseded` during upgrade and cannot be accidentally applied.
- Previously applied changes remain auditable and reversible through existing sweep history when their expected hashes still match.

## Controls

The Obsidian Vault page now shows:

- Adaptive model status, model name, last-learned time, summary, and learned note patterns.
- Candidate notes per AI decision (5–50).
- Minimum AI relationship confidence (0.50–1.00).
- Maximum evidence-backed links per note (1–50), used only as a safety ceiling.
- Relationship descriptions and source/target evidence in Review before approval.

Maintenance recommendations require Local AI. Index-only sweeps remain available without AI.

## Verification

The release test suite covers:

- A corpus of 100 near-identical invoices where one invoice links only to its named company, client, and explicit invoice index—not to other invoices.
- Rejection of invented targets, vague relationship labels, missing source/target evidence, low confidence, and ungrounded evidence.
- Complete exclusion of generated blocks from subsequent indexing and matching.
- Safe cleanup of an old 50-link generated block with user-authored content preserved.
- Safe behavior when every Local AI decision fails.
- Migration of old pending recommendations.
- A novel field-research vault taxonomy, demonstrating that categories are learned rather than compiled into the app.
- A 5,000-note adaptive-index stress test.
- Local and Desktop-hosted vault sweep, Review, apply, concurrent-edit, Stop, and Undo paths.

## Upgrade note

After upgrading, run an **Index Sweep**, then run a **Maintenance Sweep** in **Send all recommended changes to Review** mode. Inspect the proposed cleanup and relationship changes before using automatic mode. The Windows Desktop component and server must both be updated to v0.14.0 for the complete release behavior.
