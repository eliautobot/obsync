# Obsync v0.18.0

Obsync v0.18.0 makes Maintenance Sweep recommendations conservative, contextual, and individually reviewable. It keeps the useful part of the adaptive engine—finding genuinely related notes—while refusing to turn dates, `Owner`, `Status`, numbers, or generic words into misleading links.

## Context-grounded links and tags

- Every link must use an exact allowed phrase from the source and the surrounding source sentence must support the target relationship.
- Dates, times, standalone numbers, hashes, metadata labels, generic words, protected Markdown, partial words, and punctuation fragments are rejected as anchors.
- Existing path-qualified wikilinks, title/alias links, and ordinary Markdown links prevent a duplicate relationship proposal.
- A relationship shown in Review must produce a real native edit; rejected or unrenderable relationships no longer inflate counts.
- Each tag needs its own reason, grounded source evidence, confidence, and an allowed value learned from the current vault. Broad or stale generated tags do not become training vocabulary.

## Accurate metadata and organization

Maintenance reparses tags, links, properties, and content from current Markdown before learning. This removes phantom values left in the stored index and separates human-authored tags from active Obsync-owned edits.

Exact-content duplicates are detected deterministically before Local AI inference and appear as review-only canonical-note decisions. Local AI may also propose moving an unlinked ordinary note to an existing folder or adding a note to a proven category hub. These organization actions are separate Review operations; they never automatically delete or merge notes. Moves are not proposed for managed notes, backlink targets, or notes with a competing content edit.

## Operation-level Review

Open a maintenance card to inspect every operation's anchor sentence, target, evidence, reason, and confidence. Clear any checkbox you do not want, then choose **Apply selected**. Only selected operations are applied, owned, recorded, and later reversed by **Undo Sweep**.

## Sweep reliability and performance

- **Stop Sweep** cancels the active model request immediately and prevents the next note from starting.
- Server startup closes orphaned queued/running/stopping sweep rows and safely resets interrupted vault-model learning.
- Anchor discovery reuses protected Markdown ranges and bounds generic phrase mining in very large append-only logs.
- Relationship prompts bound source metadata and the highest-ranked candidate excerpts so large notes fit local-model context windows and complete reliably.
- The Review confidence meter no longer triggers a browser Content Security Policy error.
- PDF and OCR dependencies were raised to patched `pypdf` 6.x and Pillow 12.x release lines.

## Upgrade behavior

Database schema 12 supersedes pending recommendations from the earlier anchor engine, enables the `organization` maintenance category for installations still using the former links/tags default, marks cached metadata for reparsing, and invalidates the prior learned vault model. Existing Markdown is not changed during upgrade. Run a new read-only Index Sweep, then a Maintenance Sweep in Review mode.

Back up both the Obsync `/data` volume and the Obsidian vault before upgrading. To roll back, restore the pre-upgrade data backup together with the v0.17.0 application or container; do not run v0.17.0 against a database already migrated to schema 12.

## Verification

The release is covered by the full Python test and coverage suite, real-model end-to-end tests,
adversarial replay of the earlier bad proposals, and a deterministic 511-note safe-copy scale sweep
covering review, selective apply, and Undo. It also passed schema migration checks, dependency and
code security scans, Docker health/login smoke tests, standalone-agent packaging, responsive
browser validation, and native Obsidian Markdown readback.

The GitHub release includes Linux and Windows command-line agents, Obsync Desktop for Windows, and `SHA256SUMS.txt` for download verification. The matching server image is published as `ghcr.io/eliautobot/obsync:0.18.0`.
