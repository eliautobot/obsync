# Obsync v0.11.0

Obsync v0.11.0 makes Local AI activity feel continuous: model output is pushed to the browser immediately, every document trace follows independently, and manual scrolling remains under the user's control.

## Highlights

- **Immediate inference updates:** Local AI now uses an authenticated server-sent event stream. Model reasoning, output, decisions, stages, and elapsed status appear as the server receives them instead of waiting for the general three-second dashboard refresh.
- **Smooth in-place rendering:** Obsync updates the changing session metadata and trace rows without rebuilding the entire Local AI page. Settings, page position, and unrelated document panels remain stable.
- **Follow the latest activity:** Every document inference panel starts in follow mode and stays at its newest model activity automatically.
- **Manual-scroll takeover:** Scrolling away from the bottom disables follow mode for only that document. New activity continues to arrive, but Obsync does not move that panel.
- **Return-to-live control:** A down-arrow appears on a panel in manual mode. Selecting it scrolls to the newest activity and re-enables follow mode.
- **Reliable stream cleanup:** Page unload, sign-out, reconnect, and gateway disconnect paths close browser EventSource connections, auto-scroll timers, and server subscriber queues.
- **Narrow-screen fit:** Local AI cards, metadata, traces, and the return-to-live control fit mobile widths without horizontal page scrolling.

## How follow mode works

Follow state belongs to each document, not to the Local AI page as a whole. With two files being reviewed at once, you can inspect earlier output in one panel while the other keeps following its live inference. Manually reaching the bottom also restores follow mode; the down-arrow provides a quick explicit return from anywhere in the trace.

The live trace remains read-only and bounded in memory. It is not stored as a chat transcript. Models that expose a separate reasoning field show it as **MODEL THINKING**; other providers still show streamed model output and Obsync's processing stages.

## Upgrade behavior

Update the server before Obsync Desktop:

1. Back up the Obsync data volume and Obsidian vault.
2. Update the server to v0.11.0 and verify `/api/v1/health` reports `0.11.0`.
3. Open **Obsidian Vault** and confirm the exact destination before starting Global Sync.
4. On Windows, download the matching v0.11.0 Desktop app, right-click it, choose **Run as administrator**, and use **Connect and install** to update or repair the installed app.

Pairings, watched folders, processing history, source files, existing Obsidian notes, review history, and Local AI settings are preserved. This release does not require a database migration.

## Validation

- 117 automated tests pass at the required 80% total coverage threshold.
- Stream tests cover immediate activity delivery, event formatting, keepalives, authentication, and deterministic cleanup.
- Gateway stress coverage opens and closes 50 sequential streams, then 25 concurrent streams, and verifies zero subscribers remain.
- Real-browser stress testing used two simultaneous document inferences with updates every 90 milliseconds. The UI displayed new activity well below the old refresh interval, kept both panels pinned independently, preserved a manually scrolled panel while its peer continued following, and restored live mode through the down-arrow.
- Repeated browser reloads and sign-out returned the server subscriber count to zero without leaked browser timers.
- Desktop and 390-pixel mobile checks found no Local AI horizontal overflow and no browser-console errors.

## Safety

The activity stream is administrator-authenticated, same-origin, memory-bounded, and read-only. Disconnect cleanup is explicit on both sides of the gateway. This release does not change source-file handling, vault ownership checks, duplicate protection, inference cancellation, or the preserved **My notes** boundary.
