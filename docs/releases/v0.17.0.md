# Obsync v0.17.0

Obsync v0.17.0 makes whole-vault maintenance behave like native Obsidian editing: approved relationships become inline wikilinks on phrases already present in a note, and approved tags join its YAML frontmatter. Maintenance no longer adds a visible Obsync block.

## Native, vault-adaptive edits

- Relationship retrieval remains only a candidate shortlist. Local AI must identify an exact target, a specific relationship, grounded source and target evidence, and an exact safe phrase already present in the source note.
- Accepted links wrap that phrase as a path-qualified `[[wikilink|existing phrase]]`; Obsync does not append a relationship list or invent anchor text.
- Accepted tags use the vault's observed vocabulary and merge into native YAML without disturbing unrelated properties.
- Frontmatter with human comments or YAML structures that cannot be changed safely is left untouched. Windows CRLF line endings are preserved.
- The default whole-vault link ceiling is eight per note, with a configurable maximum of twenty. It is a ceiling, not a target.

## Read-only indexing and owned maintenance

Index Sweep is now unconditionally read-only. It catalogs the vault and learns its observed folders, tags, and category hubs without invoking Local AI, proposing changes, or writing Markdown.

Maintenance stores ownership for every inline link and frontmatter tag operation. This lets ordinary source synchronization rebase approved maintenance onto regenerated managed notes without treating the entire note as machine-owned. If a person removes an owned link or tag, Obsync records that change and does not resurrect it during later sync. Legacy visible maintenance blocks are excluded from evidence and can be removed by a new Maintenance Sweep.

## Review, automatic mode, and Undo

Review cards now show native operation counts, exact anchors and targets, evidence, confidence, and the complete before/after diff. Individual or bulk approval applies the same validated operations used by automatic mode. Concurrent-edit checks remain in force, and Undo Sweep reverses only still-current results while updating operation ownership.

## Upgrade behavior

The database migration supersedes pending recommendations created by the old block-based engine, forces Index Sweep back to read-only mode, and changes the legacy default link ceiling from twenty to eight. Existing notes are not rewritten during upgrade; changes occur only through a later approved or automatic Maintenance Sweep.

## Verification

This release is covered by unit and integration tests for native link/tag edits, YAML and CRLF preservation, partial-word anchor rejection, operation ownership, ordinary-sync rebasing, human-removal handling, review/automatic apply, migration, concurrent edits, and Undo. Opt-in end-to-end tests also exercised global sync and native maintenance against a real local 27B model, and generated output was parsed and found through Obsidian's own REST API.
