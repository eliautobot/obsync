# Obsync v0.6.0

This release makes first-time setup substantially easier and explains the application directly inside the interface.

## Highlights

- **No persistent PowerShell window:** the new Windows Companion installs for the current user, runs silently in the background, and starts automatically at Windows sign-in.
- **No Administrator access required:** download the Companion, enter the server address and one-time pairing code, and click **Connect and install**.
- **Built-in Help center:** a five-step quick start, page-by-page guide, status-color explanations, model setup, safety behavior, and troubleshooting are now available from the sidebar or top-right `?` button.
- **Contextual explanations:** small `?` controls explain settings, states, computer roles, folder actions, account fields, and document columns on hover or keyboard focus.
- **Clear notifications:** notices now render above dialogs and stay sharp while a modal is open.

## Windows Companion workflow

1. Open **Sources → Add another computer**.
2. Name the computer and create a pairing code.
3. Download and open `obsync-companion-windows-x64.exe` on that Windows PC.
4. Enter the displayed server address, code, and computer name.
5. Optionally select the Obsidian vault, then click **Connect and install**.

The Companion is installed under the current user's Local AppData folder. It creates a limited, per-user Windows Task Scheduler entry and starts without a visible terminal. Existing command-line agents remain supported for Windows, Linux, macOS, NAS, and advanced deployments.

Early community builds are not code-signed. If Windows SmartScreen identifies the Companion as an unrecognized app, first verify that it came from the official Obsync GitHub release, then choose **More info → Run anyway**.

## Upgrade notes

Update the Docker server first, then install the v0.6.0 Companion on each Windows desktop. Existing server data, accounts, pairings, watched folders, and generated notes are preserved. The Companion may reuse an existing pairing when repairing automatic startup.

See the [README](../../README.md) for ready-to-copy update commands and [Updating and rollback](../UPDATING.md) for backup and rollback procedures.
