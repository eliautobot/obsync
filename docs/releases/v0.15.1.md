# Obsync v0.15.1

Obsync v0.15.1 gives slower local models enough time to complete vault intelligence work and makes every timeout failure explicit and actionable.

## Configurable Local AI timeout

**Local AI → Model timeout (seconds)** now controls the maximum duration of each inference request, including:

- Document classification and organization
- Adaptive vault-model learning
- Each Maintenance Sweep relationship decision

The default is now **600 seconds** instead of 120 seconds. The setting accepts values from 5 through 3,600 seconds. A sweep can issue multiple sequential AI requests, so this value is a per-request limit rather than a total sweep duration.

Upgrades that still use the former 120-second default move to 600 seconds automatically. Any other custom timeout remains unchanged.

The quick **Check connection** action continues to use a separate 15-second limit because it only lists available models and does not run inference.

## Actionable timeout failures

Timeout messages now state:

- How many seconds elapsed
- Which operation timed out, such as adaptive vault-model learning or relationship analysis for a specific note
- Where to change the timeout
- Practical alternatives such as using a faster model or reducing an AI profile's input/output limits

The same message is retained in Maintenance Sweep history, adaptive vault-model status, activity events, and connection-check results. Empty timeout errors are no longer possible. If every per-note relationship decision fails, the final sweep failure includes the underlying Local AI error instead of only reporting that no recommendations were generated.

## Verification coverage

This release adds regression tests for:

- The new 600-second default
- Saving a custom timeout and rejecting out-of-range values
- Migrating the legacy 120-second default without changing custom values
- Explicit adaptive vault-model timeout messages
- Explicit connection-check timeout messages
- Persisted Maintenance Sweep and vault-model failure details

All existing document processing, whole-vault indexing, reconnect, review, rollback, and evidence-backed relationship tests remain part of the release gate.
