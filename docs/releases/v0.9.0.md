# Obsync v0.9.0

Obsync v0.9.0 makes vault selection explicit, separates the major controls into focused pages, and adds safer duplicate handling and independent folder controls.

## Highlights

- **Dedicated Obsidian Vault page:** Vault settings now live under **Sources → Obsidian Vault** with a prominent required choice. A new or upgraded server stops Global Sync until the destination is explicitly confirmed. Selecting the hidden `.obsidian` directory is rejected.
- **Dedicated Local AI page:** Model connection, confidence threshold, custom organization instructions, candidate-title context, and duplicate policy now live together. Custom instructions refine the protected system prompt but cannot replace its safety and JSON rules.
- **Safer existing-note matching:** Obsync continues to adopt exact managed notes by source identity and now also scans all Markdown note titles. Strong title matches are held as **Possible duplicate** before any second note is written. The user can inspect the match and explicitly choose **Create separate note**.
- **Per-folder controls:** Every watched folder has independent **Start**, **Pause**, and **Stop** controls. These cancel or resume only that folder; **Start Global Sync** and **Stop Global Sync** still control the entire installation.
- **Immediate recovery:** Starting Global Sync or restarting an individual folder queues inventory and pending-file reconciliation immediately.
- **Live interface:** Overview, Sources, Documents, Review, and application Settings update automatically every three seconds while they are visible. Forms and open dialogs are not interrupted.
- **Application-only Settings:** The Settings page now focuses on server/Desktop connection checks, opening Obsync Desktop through `obsync://`, downloading or repairing Desktop, account settings, and Help.
- **Clearer Windows setup:** The connection flow prominently instructs users to right-click Obsync Desktop and choose **Run as administrator**. Setup registers automatic startup and the `obsync://` app link; the installed watcher still runs with limited permissions.

## Local AI and Obsidian access

The Local AI model does **not** receive Obsidian API access. Obsync itself indexes the explicitly selected vault through the server mount or Obsync Desktop. The model receives extracted source text and, only when enabled, an allowlist of candidate note titles for related-note suggestions.

Duplicate protection is deliberately conservative. Managed notes are matched exactly by Obsync metadata. Unmanaged notes are held only when their normalized titles strongly match; Obsync does not silently merge them or claim that differently titled content is identical.

## Upgrade behavior

Update the server before Obsync Desktop. After the server starts:

1. Open **Obsidian Vault** and verify the exact displayed destination.
2. Save and confirm that vault.
3. Choose **Start Global Sync**.
4. On Windows, download the matching v0.9.0 Desktop app, right-click it, choose **Run as administrator**, and use **Connect and install** to update or repair the existing installation.

Pairings, watched folders, processing history, source files, and existing Obsidian notes are preserved. Upgraded installations are intentionally stopped until the vault is reconfirmed so an old or generic Docker mount cannot receive new notes by accident.

## Safety

Pause, Stop, Remove, and Disconnect never edit or delete source files. Possible duplicates are held before vault writing. Obsync continues to preserve manual text below **My notes** and refuses to overwrite unmanaged destination files.
