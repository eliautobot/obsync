# Obsync v0.16.0

Obsync v0.16.0 brings the complete live Local AI inference experience directly into Index and Maintenance Sweep panels, so administrators can follow whole-vault intelligence work as it happens.

## Live sweep inference

The **Obsidian Vault** page now displays a dedicated live AI session beneath each sweep's progress and Stop controls. During AI-assisted sweeps it shows:

- The current note and sweep phase
- Provider, model, active AI profile, and elapsed time
- Obsync processing stages
- Provider-emitted model reasoning and streamed output
- Validated relationship/tag decisions and skipped-note errors
- The final completed, stopped, or failed outcome

The most recent Index and Maintenance Sweep AI sessions remain visible after the sweep ends, matching the behavior of the Local AI page. Index-only sweeps clearly state that they do not use AI, and AI-assisted sweeps distinguish the initial indexing phase from later vault-model learning and per-note analysis.

## Same trace controls as Local AI

Sweep panels reuse the same read-only inference component as ordinary document processing:

- Each trace follows new output independently by default.
- Scrolling upward switches only that trace to manual mode.
- A down-arrow returns the trace to the latest activity.
- Live updates arrive through the existing authenticated server-sent event stream without rebuilding the page or moving the user's scroll position.

Sweep inference is intentionally not given a second cancellation button. **Stop Sweep** remains the single safe control: it prevents another note from starting after the current model request finishes or reaches its configured timeout. The trace explains this boundary while a stop is pending.

## Bounded and private

Sweep inference stays in authenticated, same-origin browser sessions. Each session retains at most 80 bounded events in server memory; it is not stored as a chat transcript. Repeated sweeps prune stale browser trace state, and disconnected browser subscribers are removed deterministically.

## Verification coverage

This release adds automated coverage for:

- Vault-model and per-note reasoning/output streams
- Completed, stopped, and failed sweep outcomes
- SSE delivery of active and most-recent sweep activity
- Index-only and pre-inference waiting states
- Validated decisions and skipped-note errors
- 2,000 rapid updates while retaining no more than 80 bounded events
- Independent follow/manual-scroll behavior and return-to-live controls
- Desktop and 390-pixel mobile layouts without horizontal overflow

All existing synchronization, reconnect, timeout, review, rollback, evidence validation, and whole-vault safety tests remain in the release gate. No database migration or vault rewrite is required for this update.
