# Obsync v0.19.0

Obsync v0.19.0 makes Maintenance links entity-specific, directional, and explainable. A shared phrase or similar topic is no longer enough: each proposed link must identify a concrete target document through a typed relationship and exact evidence from both notes.

## Private Vault Knowledge Graph

Index Sweep remains read-only, but it now prepares a private graph of the vault's canonical documents, aliases, stable identifiers, folder membership, existing links, and durable entities. Entity frequency is measured across the vault so repeated words and common concepts carry less weight than distinctive names and identifiers.

Local AI learns the vault's observed entity types, relationship types, canonicalization rules, category structure, and low-value patterns before it evaluates individual notes. This stays adaptive to each vault instead of imposing a fixed taxonomy.

No external graph service or separate graph database is required. The graph context is derived from Obsync's existing private index and the configured Local AI provider.

## Typed, evidence-backed links

Every proposed link must form a specific edge:

```text
source entity -> directional predicate -> target document entity
```

The proposal must also include an exact natural anchor in the source, the supporting source context, target-side evidence, and sufficient confidence. Broad predicates such as `related_to`, `links_to`, `reference`, and `same_as` are rejected because they do not explain why two records belong together.

The same eligibility gate now covers both Local AI results and deterministic checklist, README, reciprocal-link, and category-hub shortcuts. A shortcut can no longer bypass the semantic standard.

## Safer anchors and clearer Review

- Generic shorthand such as `backup plan` or `account inventory` is rejected when it omits the distinctive identity of the target document.
- Complete names such as `Workspace Backup Plan` remain eligible when both notes support the relationship.
- Recurring action boilerplate—draft-review instructions, port-exposure steps, keep-alive goals, and must-perform phrases—is rejected as link text.
- Review displays the proposed graph edge alongside the anchor, target, evidence, reason, and confidence.
- Deterministic cleanup of marked Obsync maintenance blocks remains separate from semantic link analysis.

## Upgrade behavior

Database schema 13 supersedes pending Maintenance recommendations created by the earlier relationship engine and invalidates the cached learned vault model. Existing Markdown is not changed during upgrade. After updating, run a new full Index Sweep, then run Maintenance in Review mode and inspect the new graph-grounded recommendations.

Back up both the Obsync `/data` volume and the Obsidian vault before upgrading. To roll back, restore the pre-upgrade data backup together with the v0.18.0 application or container; do not run v0.18.0 against a database already migrated to schema 13.

## Verification

The release is covered by the full Python test and coverage suite, package and static validation, and a deterministic 511-note safe-copy scale sweep covering Review, selective apply, and Undo. Regression coverage includes graph construction, relationship eligibility, AI and deterministic shortcut enforcement, generic and boilerplate anchors, migration behavior, operation serialization, and Review rendering.

The GitHub release includes Linux and Windows command-line agents, Obsync Desktop for Windows, and `SHA256SUMS.txt` for download verification. The matching server image is published as `ghcr.io/eliautobot/obsync:0.19.0`.
