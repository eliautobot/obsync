# Obsync v0.13.0

Obsync v0.13.0 upgrades document ingestion into a whole-vault Obsidian record-keeping system.

## Whole-vault intelligence

- Indexes complete Markdown content plus paths, folders, titles, aliases, headings, YAML properties, YAML/inline tags, outgoing links, backlinks, named entities, stable account/application/invoice/permit/project identifiers, hashes, and modification times.
- Searches every indexed note for each source document, then supplies bounded content from the most relevant notes to the configured model.
- Combines deterministic entity relevance with model-selected relationships and validates every path-qualified wikilink against the real vault.
- Raises the Full document transfer defaults to 200 ranked candidates and 100 validated links, with custom profiles supporting up to 250 links.

## Existing-note updates instead of duplicates

- Obsync identity and source hashes still provide exact managed-note matching.
- Ordinary notes can now match by exact normalized content, title, alias, stable record identifier, entity overlap, and full-text similarity.
- Exact matches are adopted automatically. Strong matches go to Review by default; ambiguous matches always require review.
- Adoption preserves the complete original note below Obsync's managed section.
- Later source changes update the same destination note in place, retaining identity and manual content.

## Index and Maintenance Sweeps

The Obsidian Vault page now includes separate Index Sweep and Maintenance Sweep controls:

- Start and Stop manually.
- Schedule daily, weekly, monthly, or at a custom hourly interval in a selected timezone.
- Choose index-only, send recommendations to Review, or automatically apply recommendations.
- Watch live note-level progress and see last-run history.
- Configure maintenance categories, relationship strictness, and the maximum validated links per note.

The combined Review page shows reason, confidence, supporting evidence, and complete before/after content. Changes can be applied or disregarded individually or in bulk.

## Safety and rollback

- Automatic mode warns that it can change existing notes without human approval.
- Every maintenance recommendation records complete before/after content and the expected source hash.
- If a person edits the note after the recommendation, Obsync rejects the stale write.
- Undo Sweep restores applied changes that are still current.
- Sweeps never automatically delete or merge notes, never enter `.obsidian`, skip symlinks, and use atomic writes.
- Source files remain read-only.

## Validation

The final release candidate passed 140 automated tests at 80% branch coverage, plus lint, formatting, JavaScript syntax, package, Docker Compose, and production-image checks. Coverage includes unit, integration, remote Desktop-vault, interruption, concurrent-edit, large-vault ranking, schedule, review, adoption, in-place update, and rollback behavior.

A real browser and Docker stress run indexed 5,000 Markdown notes in 14.7 seconds. Manual Stop safely interrupted a separate 5,000-note rebuild after 4,290 notes without replacing the previous complete index; a scheduled 5,000-note maintenance sweep also completed in Review mode without unapproved writes. Desktop and mobile layouts had no page-level overflow or clean-session console errors.
