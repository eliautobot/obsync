# Obsync v0.7.0

Obsync v0.7.0 makes multi-computer setup recoverable and keeps the Sources page clean.

## Highlights

- **Disconnect old computers:** Every paired desktop now has a safe Disconnect action. It revokes that computer immediately and removes its Obsync folder/file ledger while preserving the original files and all existing Obsidian notes.
- **No duplicate pairing failures:** Only one Companion setup window can pair at a time. Reopening the Companion reuses a valid saved connection instead of consuming the one-time code again.
- **Reliable recovery:** Registration retries are idempotent, automatic startup is verified, and reopening the Companion repairs the Windows startup task.
- **Less typing:** Copy all server, code, and computer-name details in one click, then paste them into the Companion with one button.
- **App-served Windows download:** Published Docker images include the matching Windows Companion, so the in-app download no longer depends on manually finding the right release asset.
- **Clear errors:** Setup failures now explain whether the server is unreachable, the saved connection was revoked, or a fresh code is required.
- **Cross-platform path safety:** Absolute paths are rejected consistently on Windows and Unix systems before any source-file operation.

## Why the Windows Companion remains necessary

A Docker container and a web browser cannot safely browse or continuously watch arbitrary Windows folders outside their mounts. The Companion is the one-time native bridge that runs as the signed-in Windows user. It needs no Administrator access, opens no inbound port, leaves no PowerShell window open, and starts automatically at sign-in.

## Safe computer removal

Disconnecting a computer:

- Revokes its device credential immediately
- Removes its paired-computer card, watched folders, file ledger, and pending commands
- Never edits or deletes source files
- Never deletes generated Obsidian notes

If the computer is the active desktop vault writer, Obsync requires choosing another vault in Settings before it can be disconnected.

## Upgrade notes

Update the Docker server first, then open the v0.7.0 Windows Companion once on every Windows desktop. It reuses the existing pairing and repairs automatic startup. Existing accounts, settings, folders, processing history, source files, and Obsidian notes remain intact.
