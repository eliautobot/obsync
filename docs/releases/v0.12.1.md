# Obsync v0.12.1

Obsync v0.12.1 fixes a timing race that could make **Start Global Sync** appear to take several minutes before work resumed.

## What changed

- **Atomic resume state:** Every Desktop command response now includes the server's current Global Sync state. Obsync Desktop applies that state before it handles queued reconciliation work.
- **Faster command pickup:** The Desktop command loop now checks once per second instead of once every three seconds.
- **No five-minute fallback delay:** A reconcile command can no longer be rejected because a separate heartbeat still carries the old stopped state.
- **Regression coverage:** The release test suite reproduces the exact failing order: stop, heartbeat the stopped state, start on the server, then receive reconcile before another heartbeat.

## Upgrade behavior

Update the server and every Obsync Desktop installation to v0.12.1. The server remains compatible with older agents, but the Desktop update is required for the resume fix and faster command pickup.

There is no database migration. Accounts, pairings, watched folders, AI profiles, source history, review decisions, vault settings, and existing notes are preserved.

## Validation

- The regression test proves reconciliation completes even when Desktop still held the old stopped state before polling commands.
- The full unit and integration suite, branch-coverage gate, linting, formatting, package builds, JavaScript syntax checks, Docker health checks, Windows and Linux CI, standalone binaries, and published container are release gates.

## Safety

This release changes only agent control-plane synchronization and polling latency. Source documents remain read-only, existing Obsidian notes are preserved, managed-note writes remain atomic, and Global Sync stop still cancels pending and active processing.
