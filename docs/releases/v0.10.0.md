# Obsync v0.10.0

Obsync v0.10.0 makes active processing visible, gives every review item a complete resolution path, and keeps large live-updating pages compact and stable.

## Highlights

- **Current active file on Overview:** Overview now shows each file being extracted, analyzed, checked, or written, including its source computer, watched folder, stage, and elapsed time.
- **Live Local AI session:** The Local AI page shows the active file, provider, model, processing stages, provider-emitted reasoning, streamed output, and final decision. It is a read-only activity view rather than a chat.
- **Stop one inference:** **Stop inference** cancels only the selected model request. Global Sync and unrelated folders remain running, the source file remains untouched, and the stopped document moves to Review.
- **Complete review decisions:** Every review item can be **Approved**, **Disregarded**, or sent through **Redo AI review**. Re-review accepts optional feedback about the desired title, category, tags, or other correction. Possible duplicates also retain **Create separate note**.
- **Stable live refreshes:** Change-aware updates preserve the page position, list-panel position, focused form, and open dialog instead of snapping the user back to the top.
- **Compact bounded lists:** Documents, Review, recent activity, active work, folder inventory, and AI traces stay inside independently scrollable panels instead of growing the entire page indefinitely.
- **Responsive document layout:** Wide document rows wrap inside the available desktop width and become readable cards on narrow/mobile screens without horizontal page scrolling.

## Local AI activity and privacy

Obsync uses streamed Ollama and OpenAI-compatible responses when supported. The live view labels provider-emitted reasoning as **MODEL THINKING**, the structured response as **MODEL OUTPUT**, and Obsync's accepted result as **DECISION**. Some models do not expose a separate reasoning field; in that case the view still shows Obsync's stages and streamed output.

The activity trace is bounded in server memory and is not persisted as a chat transcript. Reviewer feedback is stored with the affected document for audit history and is sent to the configured model only for that requested re-review. The model still has no Obsidian API access: Obsync reads the explicitly selected vault and can provide only the allowed candidate note titles.

## Review behavior

- **Approve** accepts the current category/tags. For a possible duplicate, it accepts the existing-note match and does not create a second note.
- **Disregard** creates no new note, never deletes an existing managed note, and removes the item from Review. Ordinary rescans keep it ignored until the user explicitly requests another AI review.
- **Redo AI review** forces the unchanged source through Local AI again and includes optional reviewer feedback under Obsync's protected safety and JSON instructions.
- **Create separate note** remains the explicit override when a possible duplicate should actually become its own note.

## Upgrade behavior

Update the server before Obsync Desktop:

1. Back up the Obsync data volume and Obsidian vault.
2. Update the server to v0.10.0 and verify `/api/v1/health` reports `0.10.0`.
3. Open **Obsidian Vault** and confirm the exact destination before starting Global Sync.
4. On Windows, download the matching v0.10.0 Desktop app, right-click it, choose **Run as administrator**, and use **Connect and install** to update or repair the installed app.

Pairings, watched folders, processing history, source files, existing Obsidian notes, and v0.9 settings are preserved. The database migration adds only review feedback and resolution metadata.

## Validation

The release is covered by unit and API integration tests for streamed model activity, cancellation isolation, review decisions, forced re-review feedback, and ignored-document persistence. Browser end-to-end validation uses a real streaming model endpoint and verifies live active-file display, stop behavior, scroll retention, review workflows, zero console errors, responsive reflow, and bounded scrolling with an 80-row document stress inventory.

## Safety

No review or stop action edits or deletes source files. Disregard writes no new note and never deletes one already present. Stop inference does not stop Global Sync. Existing Obsidian notes remain protected by Obsync's managed-note ownership checks and preserved **My notes** boundary.
